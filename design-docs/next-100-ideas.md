# Next 100 ideas (editor-inspired, design-crafted)

A second batch of 100 ideas, generated by a 12-lens multi-agent pass (Game feel,
Figma, After Effects, Music DAW, Blender/DCC, Code editor, Photo editor, Pro
video, Presentation, Whiteboard, Keyboard-first, a11y/design-system), deduped
against [the first top-100 list](top-100-ideas.md) and ranked by value vs effort.
Each idea is grounded in real code (`ThreeWheelRenderer.playSpin`, the `onTick`
stream, `playTick`/`playThunk`, `updateInPlace`, `previewOverrides`, the flat
~14-button dock, `ROW_H=34` virtualization, the single polite `aria-live`, and
the lack of any `:focus-visible` system).

Effort: S = small, M = medium, L = large. Value: 1-5.

---

## Implemented this session

Shipped the high-confidence, frontend-only, low-risk core of the shortlist plus a
genuine latent bug fix. All verified by `vue-tsc` + `vite build` + 31/31 vitest +
clean mount with zero console errors. The 3D *feel* of the renderer items could
not be visually verified here (the preview tab is hidden so rAF is paused, and no
backend is running to trigger a real spin) — they are code-complete and
type-checked, pending a real-browser pass.

- **Tick bug fix + rising-pitch tick ramp** — `spinSegmentIds` was declared but
  *never populated*, so `this.spinSegmentIds[crossed]` was always `undefined` and
  the spin played **no peg ticks at all**. Now populated from `segmentMeshes`
  order at spin start, and `playTick(progress, boost)` ramps 660Hz -> 1200Hz with
  spin progress and swells gain on the slow-mo tail.
  ([ThreeWheelRenderer.ts](../frontend/src/renderers/ThreeWheelRenderer.ts))
- **Hitstop on the winning peg** — a ~70ms freeze ~0.5deg past the resting angle
  before `playThunk`/reveal, then a settle onto target; skipped under
  reduced-motion. ([ThreeWheelRenderer.ts](../frontend/src/renderers/ThreeWheelRenderer.ts))
- **Anticipation wind-up squash** — the wheel compresses to 0.97 at launch and
  the frame loop eases it back to 1.0 over the first ~12%; reduced-motion skips it.
  ([ThreeWheelRenderer.ts](../frontend/src/renderers/ThreeWheelRenderer.ts))
- **Unified focus-visible ring token** — one `--accent` + `:focus-visible`
  outline rule in [style.css](../frontend/src/style.css), a `:focus:not(:focus-visible)`
  reset, and a high-contrast override; the four bare `outline:none` overrides
  (App, NameList x2, CommandPalette) now inherit the shared ring.
- **j/k history nav with ghost preview** — roving keyboard selection in
  [HistoryTimeline.vue](../frontend/src/components/HistoryTimeline.vue) reusing the
  existing `preview`/`restore` emits; j/k (or arrows) ghost-preview each snapshot,
  Enter restores, Escape clears, with scroll-into-view and listbox a11y.
- **Undo toast draining bar** — a hover-pausable countdown along the bottom edge
  of undo toasts (mirrors `DEFAULT_TTL_MS`), making the invisible undo window
  legible. ([ToastStack.vue](../frontend/src/components/ToastStack.vue))

Already present from the prior batch (so not re-done): the **lower-third build-in
sequence** (`SpinResult.vue` already has pop-settle + accent sweep +
reduced-motion fallback).

Deferred as next-tier (larger surface area I cannot visually verify here, or
higher render/netcode risk): live velocity ribbon, transform HUD, Magic-Move
re-balance tween, keystroke pulse on dock controls, empty-palette suggested-next,
and all multiplayer/lighting/camera-rig items.

## Curation notes (what was cut and why)

The 12-lens pass produced 145 raw ideas; these were deduped against the existing
top-100 and collapsed across lenses so each survivor is distinct:

- **History-as-scrubber/playhead/filmstrip/time-remap** appeared ~8 times across
  AE / Pro-video / Photo / Code / Presentation / Whiteboard and overlaps the
  existing 'history mini thumbnails' + 'snapshot scrub'. Kept In/Out markers
  (#50), j/k+ghost (#26), loop brace (#90), and history detents (#75); cut the
  rest as redundant reskins.
- **Real multiplayer cursors** appeared as Figma cursors, peer halos, co-editor
  echo, live carets, playhead presence flags. Kept #77 (cursors+flags) and #97
  (follow-camera); cut the rest.
- **Direct-manipulation weight family** was large; kept the distinct set (scrubby
  label, scrub-on-wheel, odds curve, macro knob, slider detents, snapping,
  eyedropper) and cut grading-wheel (redundant with curve+scrubby) and graduated
  filter (gimmicky, low value, high risk).
- **Cut as already-covered by the existing list:** autoplay rehearsal,
  phone-as-remote, unlockable achievements, weight-expression pill (eval risk),
  ripple-delete, snap-to-edge draggable dock, remote director's keyframe cursor.

Honesty caveat from the curator: 'authored motion-blur shutter' (#17) and
'laser-pointer trail' (#93) assume a motion-blur ring already exists; that was not
re-verified in code, so those two lean on the brief's claim.
